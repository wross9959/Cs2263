Processing strings, arrays on the heap

Develop and test a new version of the htags program from Assignments 2 and 3 for examining the HTML tags in an HTML file, called �htags3�. The htags3 program should work similarly to htags2 program except it should allocate storage on the heap instead of declaring arrays with fixed size on the call stack. The detected tags are to be stored in a 2D array allocated on the heap, and the size of the array, at any given time, must be no greater than what is required for tags read so far. Every time new memory is allocated on the heap, the program should print the amount of the new allocation (in bytes) and the total memory allocated so far. Finally, before the program ends, you must deallocate (free()) all the storage used on the heap.

All other requirements stay the same: specify the name of the html file as a command line parameter, then process this file printing each tag only once.  The printout should show both the tags and the memory allocation figures.
